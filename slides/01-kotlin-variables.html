Kotlin Variables: labeled jars.

By the end: you know when to seal a jar (val) vs give it a lid (var) — and can say it like a pro.

🧒 The simple idea

A variable is a labeled jar

You write a value on a labeled jar so the computer can find it again by its label. No jars = the program can't remember anything.

Core concept

Two kinds of jar

  • val — sealed jar 🔒: set once, never changes. Your default.
  • var — jar with a lid 🔄: can be reassigned later.
  • Trying to change a val is a compile error — a feature that catches bugs.
💻 In code

See it work

fun main() {
    val name = "Aiyappa"   // sealed jar (String)
    var age = 25           // lidded jar (Int)
    age = 26               // OK — age is a var
    println("Hi $name, you are $age")
}
// prints:  Hi Aiyappa, you are 26
🎤 Q&A recall

Say it like a pro

  • Q: val vs var?
  • A: "val is read-only — assigned once, can't be reassigned. var is mutable. I default to val for immutability to prevent accidental state changes."
  • Q: What does $ do? A: "String interpolation — inserts a variable's value into text."
Recap

What we locked in

  • Variable = labeled jar that remembers a value.
  • val = sealed (default) · var = lidded (only when needed).
  • $ = string interpolation. Next: types, operators & input.
